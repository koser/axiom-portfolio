# Interval Drift
**Date:** 2026-03-15
**Project ID:** AG04
**Library:** p5.js

## Concept
A study in perceptual instability through vertical stripe intervals. What appears stable reveals micro-shifts in spacing and alignment, creating a field that breathes without obvious motion. The piece exploits peripheral vision's sensitivity to pattern disruption — stare at the center and the edges seem to crawl.

## Palette
#1a1a1a (deep background), #c4c4c4 (primary stripes — neutral mass), #2d2d2d (secondary stripes — dark intervals), #e8e8e8 (accent — rare bright interruptions)

## Composition
Full-width vertical stripe field, 50 stripes spanning viewport height. Initial state: stripes distributed in three horizontal bands — left band (x: 0 to width*0.3) contains 18 stripes at 12-18px intervals, center band (x: width*0.3 to width*0.7) contains 20 stripes at 8-14px intervals, right band (x: width*0.7 to width) contains 12 stripes at 20-30px irregular intervals. Phase 1 (0-20s): bands maintain distinct densities while individual stripes oscillate within their zones. Phase 2 (20-55s): band boundaries dissolve, stripes from left band migrate right at 0.8-1.4px/frame creating diagonal drift pattern from lower-left to upper-right (y-dependent velocity: vy = map(y, 0, height, 0.8, 1.4)). Phase 3 (55-120s): reverse migration, all stripes spiral toward original positions via polar coordinates — angular velocity 0.012-0.025 radians/frame, radial compression 0.6px/frame toward baseX.

## Elements
Vertical stripes, each 6-16px wide (width varies by band: left band 12-16px, center band 6-10px, right band 10-14px), full viewport height (y: 0 to windowHeight). Total count: 50 stripes. Primary stripes (#c4c4c4): 32 stripes (64%). Shadow stripes (#2d2d2d): 14 stripes (28%), render first in draw order. Accent stripes (#e8e8e8): 4 stripes (8%), pulse at 0.6-1.0 opacity over 120-180 frame cycles rather than flash. Each stripe stores: baseX (original x), x (current x), y (top position, always 0 for vertical stripes but used for height segments), baseWidth (6-16px based on band), width (current width modulated), vx (horizontal velocity px/frame), vy (vertical drift for height segments), angle (current rotation in radians, 0 for Phase 1-2, active in Phase 3), angularVelocity (0.012-0.025 radians/frame), phaseOffset (0-6.28 radians random), band (0=left, 1=center, 2=right), type (primary/shadow/accent), noiseOffset (random 0-2000 for Perlin noise lookup), heightSegments (array of 8 segments per stripe, each tracking independent y-offset for Phase 2 wave distortion).

## Animation
Phase 1 (0-1200 frames / 0-20s): Stripes oscillate within band constraints. Left band: x = baseX + sin(frameCount*0.008 + phaseOffset)*15 + noise(noiseOffset + frameCount*0.002)*8, amplitude ±15px sine, ±8px noise. Center band: x = baseX + sin(frameCount*0.015 + phaseOffset)*6 + noise(noiseOffset + frameCount*0.003)*4, tighter oscillation ±6px sine, ±4px noise. Right band: x = baseX + sin(frameCount*0.005 + phaseOffset)*25 + noise(noiseOffset + frameCount*0.001)*12, slower wider drift ±25px sine, ±12px noise. Width pulses all bands: width = baseWidth + sin(frameCount*0.02 + phaseOffset)*3, amplitude ±3px, period 314 frames (5.2 seconds). Phase 2 (1200-3300 frames / 20-55s): Diagonal migration activated. All stripes apply diagonal drift: vx = 0.8 + (stripe.index * 0.012), rightward velocity increases by stripe index (first stripe 0.8px/frame, last stripe 1.4px/frame). Height segments introduce vertical wave distortion: each stripe divided into 8 vertical segments (windowHeight/8 per segment), each segment applies independent y-offset = sin(frameCount*0.018 + phaseOffset + segment*0.6)*20, creating vertical ripple amplitude ±20px propagating down stripe, phase offset between segments 0.6 radians. Width modulation increases: width = baseWidth + sin(frameCount*0.025 + phaseOffset)*7 + cos(frameCount*0.012)*4, dual-frequency interference amplitude ±11px peak. Band boundaries fade: stripes no longer constrained, wrap at edges (if x > width+50, x = -50). Phase 3 (3300-7200 frames / 55-120s): Spiral return to origin. Each stripe calculates polar coordinates relative to baseX: deltaX = x - baseX, distance = abs(deltaX), angle = (deltaX > 0 ? 0 : PI). Angular velocity applied: angularVelocity = map(stripe.index, 0, 50, 0.012, 0.025) radians/frame, stripes rotate counter-clockwise if currently right of baseX, clockwise if left. Radial compression: distance *= 0.988 per frame (1.2% decay), x = baseX + cos(angle)*distance. Vertical segments collapse: y-offsets decay by *= 0.92 per frame toward 0. Width stabilizes: width = baseWidth + sin(frameCount*0.008 + phaseOffset)*1.5, reduced to ±1.5px amplitude. Every 360 frames (6 seconds), select 6-10 random stripes, apply sudden 180-degree angular flip (angle += PI) over 1 frame, creating staccato disruptions in spiral pattern. Accent stripes: continuous sine-wave pulsing opacity = 0.6 + sin(frameCount*0.031 + phaseOffset)*0.4, oscillating 0.6-1.0 over 203 frames (3.4 seconds).

## Transformation
Phase 1: Width modulation via single sine wave: width = baseWidth + sin(frameCount*0.02 + phaseOffset)*3. Horizontal position oscillates: x = baseX + sin(frameCount*bandFreq + phaseOffset)*bandAmplitude + noise(noiseOffset + frameCount*bandNoiseSpeed)*bandNoiseAmplitude, where bandFreq/Amplitude/NoiseSpeed vary by band (left: 0.008/15/0.002, center: 0.015/6/0.003, right: 0.005/25/0.001). Phase 2: Width dual-frequency: width = baseWidth + sin(frameCount*0.025 + phaseOffset)*7 + cos(frameCount*0.012)*4. Horizontal velocity accumulates: x += vx, where vx = 0.8 + (stripe.index * 0.012). Vertical segments: 8 segments per stripe, segment[i].yOffset = sin(frameCount*0.018 + phaseOffset + i*0.6)*20, each segment renders rect(x, i*segmentHeight + yOffset, width, segmentHeight). Wrapping: if x > width+50, x = -50; if x < -50, x = width+50. Phase 3: Polar coordinate transformation: distance = abs(x - baseX), angle += angularVelocity (0.012-0.025 radians/frame based on stripe index), distance *= 0.988, x = baseX + cos(angle)*distance. Vertical segment offsets decay: segment[i].yOffset *= 0.92 per frame. Width stabilizes: width = baseWidth + sin(frameCount*0.008 + phaseOffset)*1.5. Angular flips every 360 frames: 6-10 random stripes execute angle += PI over 1 frame. Mouse influence (all phases): vx += mouseInfluence.driftForce, width += mouseInfluence.widthMod. Opacity: primary/shadow always 1.0, accents pulse 0.6-1.0. No rotation in Phase 1-2, active rotation in Phase 3. Draw order: shadow stripes first, primary stripes, accent stripes last.

## Rhythm
Macro: 120-second evolution from zoned oscillation (Phase 1, 20s) through diagonal dissolution (Phase 2, 35s) to spiral reconvergence (Phase 3, 65s). Phase transitions blend over 240 frames (4 seconds at 60fps) using easeInOutQuad interpolation. Micro: each stripe has unique oscillation period creating polyrhythmic field. Phase 1: 314-frame cycles (5.2 seconds) per stripe with band-specific frequencies (left: 0.008, center: 0.015, right: 0.005 Hz equivalent) = periods of 785, 419, 1257 frames. Phase 2: diagonal drift introduces index-based phase spread — stripes enter migration sequentially over 600 frames (10 seconds), creating staggered diagonal wave. Vertical segment waves: 8 segments * 0.6 radian offset = 4.8 radian total phase spread per stripe, segments oscillate at 349-frame period (5.8 seconds). Phase 3: spiral periods vary by stripe 254-523 frames (4.2-8.7 seconds, based on angularVelocity 0.012-0.025 rad/frame to complete one revolution). Angular flip shockwaves every 360 frames (6 seconds) affecting 6-10 stripes. Accent pulse rhythm: 203-frame cycles (3.4 seconds), four independent accents create beating patterns every 50-200 frames.

## Interaction (mouse)
Mouse X position (0 to windowWidth) mapped to horizontal drift bias: driftForce = map(mouseX, 0, windowWidth, -0.8, +0.8) px/frame, applied as vx += driftForce to all stripes. Left edge (mouseX=0): -0.8px/frame leftward pull. Right edge (mouseX=windowWidth): +0.8px/frame rightward pull. Center (mouseX=windowWidth/2): 0 force. Mouse Y position (0 to windowHeight) controls width modulation additive: widthMod = map(mouseY, 0, windowHeight, -5, +5) px, applied as width += widthMod after base calculation. Top (mouseY=0): -5px width reduction (thinner stripes). Bottom (mouseY=windowHeight): +5px width increase (thicker stripes). Center (mouseY=windowHeight/2): 0 modification. Mouse click (mousePressed): trigger angular shockwave from click position. Calculate polar angle from center: clickAngle = atan2(mouseY - height/2, mouseX - width/2). All stripes within 45-degree cone (abs(stripe.angle - clickAngle) < PI/4) and distance < 400px from click receive angular impulse: angularVelocity += 0.08 radians/frame for 60 frames (1 second), then decay back to base velocity over 120 frames using linear interpolation. Creates localized spiral acceleration in clicked region. Mouse drag (mouseIsPressed && abs(mouseX-pmouseX) > 2): vertical segment manipulation. Stripe closest to mouseX has its segments pulled: segment[i].yOffset += map(mouseY-pmouseY, -50, 50, -15, 15) * map(i, 0, 7, 1.0, 0.3), upper segments respond more than lower segments. Effect persists for 180 frames after drag ends, then decays via *= 0.94 per frame. Mouse idle detection: if no movement (abs(mouseX-lastMouseX) < 1 && abs(mouseY-lastMouseY) < 1) for 240 frames (4 seconds), decay mouse forces over 180 frames: driftForce *= 0.97 per frame, widthMod *= 0.97 per frame until both reach 0.1px threshold then snap to 0.
