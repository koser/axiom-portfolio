# Woven Loop Grid
**Date:** 2026-03-16
**Project ID:** AG18
**Library:** p5.js

## Concept
The moodboard shows a 2x2 grid of four identical compositions, each containing a continuous black stroke that weaves through a regular 4x4 grid of light gray circles. The stroke forms organic, rounded rectangular shapes with varying loop patterns, creating a maze-like structure. When animated, each stroke becomes a living thread that slowly traces and retraces its path with breathing motion in the loops. The gray circles pulse in response to the stroke's proximity, and the composition exhibits gentle morphing where the stroke's bezier control points shift along constrained perpendicular vectors, maintaining the overall woven topology while introducing organic variation. The piece explores the tension between rigid grid structure and organic flow, between perfect repetition and subtle deviation.

## Palette
#f5f5f5 (canvas background - warm off-white), #1a1a1a (primary stroke - deep charcoal black), #d8d8d8 (grid circles - cool light gray), #0a0a0a (stroke shadow layer for depth), #e8e8e8 (circle highlight state during activation)

## Composition
Canvas: 800x800px. Four identical quadrants in 2x2 grid, each 380x380px with 20px gutter between them and 10px margin from canvas edge. Each quadrant contains a 4x4 grid of 16 circles with 38px spacing between centers (measured from first to last: 3 intervals × 38px = 114px span, centered in quadrant leaving ~133px on each side). Circle specifications: 30px diameter, 1.5px stroke weight, no fill. The woven stroke path overlays this grid, constructed from 36 bezier control points forming a continuous non-self-intersecting loop that encloses certain circles in rounded rectangular formations while weaving linearly between others. Path entry point: left edge at approximately 40% height. Stroke weight: 2.8px with ROUND caps and joins. Corner radius: 20px default. Quadrant animation phases offset: top-left 0s, top-right 0.75s, bottom-left 1.5s, bottom-right 2.25s, creating diagonal cascade.

## Elements
PRIMARY WOVEN STROKE: Constructed using custom bezier path with 36 control points. Each control point stored as object {x: float, y: float, handleIn: {x, y}, handleOut: {x, y}, baseX: float, baseY: float}. Path topology: enters from left edge, creates 2-3 rounded rectangular enclosures (radius 20px) around circle clusters, weaves between remaining circles with smooth curves, exits and loops back to entry forming closed path. Total path length: ~850-950px. Rendered using beginShape(), bezierVertex() for each segment. Primary render: #1a1a1a, weight 2.8px. Shadow layer: same path offset +1.2px x, +1.2px y, color #0a0a0a alpha 0.18, weight 2.8px, rendered first. GRID CIRCLES: 64 total circles (16 per quadrant). Each circle object {baseX, baseY, diameter: 30, strokeWeight: 1.5, opacity: 0.7-1.0, scale: 1.0-1.15, fillAlpha: 0.0-0.05}. Base color #d8d8d8. Circles pulse in opacity based on nearest stroke distance: within 0-25px maps to opacity 1.0, 25-50px maps to 0.85-1.0, beyond 50px rests at 0.7. Render with noFill() by default, fill(#1a1a1a, fillAlpha * 255) when mouse-activated. MORPHING DISPLACEMENT FIELD: Each control point has associated displacement vector perpendicular to local path tangent. Displacement calculated using 3-octave Perlin noise: displacement = noise(pointIndex × 0.08 + time × 0.00015) × 6px for primary oscillation, plus traveling wave component: sin(time × 0.0008 + pointIndex × 0.15) × 3px. Total displacement range: ±9px maximum. CORNER RADIUS MODULATION: Each bezier curve's corner tightness pulses between 15-25px using sinusoidal function with 55-second period, phase-offset by control point index creating traveling wave of corner looseness along path at 15px/second.

## Animation
PHASE 1 - EMERGENCE (0-10s): Stroke path draws progressively from starting control point using dash offset animation or incremental vertex rendering. Drawing speed: 95px/second (total path ~950px / 10s). Line grows smoothly with ease-in-out-cubic easing function applied to drawing speed (slower at start/end, faster in middle). As stroke tip passes within 35px of each circle, that circle fades in from opacity 0 to 0.7 over 0.4s (exponential ease-out). At 10s mark, full path visible, all circles at base opacity 0.7. Hold static for 2s. PHASE 2 - BREATHING (12-45s): Control point displacement activates gradually, amplitude ramping from 0 to full 9px over 3-second transition (12-15s). Primary breathing frequency: 0.018 Hz (55-second cycles). Each control point oscillates along perpendicular vector with amplitude modulated by multi-layered noise: base noise (0.018 Hz) × traveling wave (0.04 Hz) × micro-jitter (0.08 Hz at 1px amplitude). Circles begin proximity-responsive pulsing: opacity updates every frame based on nearest stroke point distance using inverse square law: opacity = 0.7 + 0.3 × (1 - pow(distance/50, 2)). Circle scale subtly responds to opacity: scale = 1.0 + (opacity - 0.7) × 0.1. Corner radii begin 55-second pulse cycles with 0-8s phase offset per control point creating wave traveling clockwise at 15px/s. PHASE 3 - SYNCHRONIZED FLOW (45s onward): Quadrant-level coordination activates. A primary wave rotates around the 2x2 grid with 20-second period (0.05 Hz). Wave position calculated as: waveAngle = time × 0.314 radians/second (20s per revolution). Each quadrant's displacement amplitude scales by 0.7 + 0.3 × sin(waveAngle - quadrantAngle) where quadrantAngle is 0, π/2, π, 3π/2 for top-left, top-right, bottom-left, bottom-right. This creates synchronized bulging: when wave passes a quadrant, all its control points displace outward by additional 4px (total 13px peak), then contract back. Micro-rhythms continue underneath at original 0.018-0.08 Hz range. Every 80 seconds: motion decelerates over 4s to held pause lasting 2.5s (only 0.5px jitter remains), then accelerates back over 3s using cubic-ease-in-out.

## Transformation
CONTROL POINT DISPLACEMENT: Each point moves along vector perpendicular to path tangent at that point. Tangent calculated from bezier derivative at control point position. Perpendicular vector = {-tangent.y, tangent.x} normalized. Displacement amount calculated per frame: baseDisplacement = noise(pointIndex × 0.08, time × 0.00015) mapped from [0,1] to [-1,1] then scaled to ±6px. Add traveling wave: waveDisplacement = sin(time × 0.0008 + pointIndex × 0.15) × 3px. Total: currentDisplacement = baseDisplacement + waveDisplacement, clamped to ±9px. New position: displayX = baseX + perpVector.x × currentDisplacement, displayY = baseY + perpVector.y × currentDisplacement. Bezier handles rotate with control point to maintain curve smoothness. CORNER RADIUS PULSE: For each bezier segment, calculate corner radius as: radius = 20 + 5 × sin(time × 0.114 + pointIndex × 0.18) where time in seconds, 0.114 rad/s = 55s period. Radius affects handle length: shorter handles = tighter corners. Handle length = radius × 0.552 (circle approximation constant). CIRCLE RESPONSE DYNAMICS: Each frame, find nearest stroke point to each circle using sampled distance checks (sample stroke at 8px intervals). Calculate distance d. Map to target opacity: targetOpacity = 0.7 when d > 50px, lerp(1.0, 0.85, (d-25)/25) when 25px < d < 50px, 1.0 when d < 25px. Current opacity springs toward target: velocity += (targetOpacity - currentOpacity) × 0.25 - velocity × 0.18, currentOpacity += velocity. Scale follows opacity: targetScale = 1.0 + (currentOpacity - 0.7) × 0.167 (maps 0.7-1.0 opacity to 1.0-1.05 scale). Spring constants: k=0.22, damping=0.16. CYCLE STRUCTURE: All noise-based animations loop seamlessly every 90 seconds (noise space advances 0.00015 × 90 × 60fps = 0.81 units). Phase offsets prevent perfect synchronization. Wave-based animations have prime-number period ratios (55s, 20s, ~8s) to create evolving interference patterns that don't perfectly repeat for 1100+ seconds.

## Rhythm
MACRO RHYTHM: Primary breathing at 0.018 Hz (55.5-second full inhale-exhale cycle). This is the fundamental period for corner radius pulsing and base displacement amplitude modulation. Quadrant synchronization wave at 0.05 Hz (20-second rotation). These two frequencies create 20:55 ratio (approximately 4:11) generating 220-second before pattern repeats. MICRO RHYTHM: Individual control point jitter at 0.08 Hz (12.5-second cycles) with 2:1 amplitude ratio to macro rhythm (1.5px vs 3px wave amplitude). Circle response time: attack 0.25s (opacity rise when stroke approaches), decay 0.6s (opacity fall when stroke recedes). Fast response creates sparkle punctuation against slow structural breathing. POLYRHYTHM STRUCTURE: Three simultaneous frequencies create complexity: 0.018 Hz (structure) : 0.04 Hz (traveling wave) : 0.08 Hz (jitter) = 9:20:40 ratio. These frequencies constructively interfere every 100 seconds, destructively interfere at 50s intervals, creating crescendo-decrescendo of visual complexity. SILENCE MOMENTS: Every 80 seconds, 4-second deceleration (cubic-ease-out reduces all velocities to 5% of normal), 2.5s held pause (only 0.5px amplitude jitter remains active), 3s re-acceleration (cubic-ease-in returns to full motion). These pauses reset perceptual attention and create dramatic re-entry. During pause, circle opacity holds at current values rather than continuing response.

## Interaction (mouse)
PROXIMITY FIELD: Mouse creates circular influence with 120px outer radius, 40px inner radius of maximum effect. Control points within this field experience attraction force toward mouse. Force calculation per point: distance d = distance from point to mouse. If d < 40px: force = 0.12. If 40px < d < 120px: force = 0.12 × pow((120-d)/80, 1.5). Force vector = normalize(mouse - point) × force × 15px = displacement. Applied additively to animation displacement, capped at total 24px from base position. When mouse velocity exceeds 10px/frame, create wake effect: control points within 140px radius experience delayed attraction with spring lag (k=0.14, damping=0.88), creating visible trailing bulge behind cursor movement direction. CIRCLE HOVER: Mouse within 30px of circle center triggers activation. Circle properties animate: targetScale = 1.15 (from 1.0), targetFillAlpha = 0.05 (from 0). Spring physics: scaleVelocity += (targetScale - currentScale) × 0.24 - scaleVelocity × 0.18. Similar for fillAlpha with k=0.28, damping=0.22. Attack time ~0.25s, decay when mouse exits ~0.5s. Activated circles render with fill(#1a1a1a, fillAlpha × 255) before stroke. CLICK WAVE PULSE: On mousePressed(), determine which quadrant contains mouse (if any). Find nearest control point on that quadrant's stroke (search all 36 points, use min distance). Create bidirectional WavePulse object: {originIndex: int, position: float (0 to pathLength), amplitude: 12px, speed: 180px/s, width: 40px, decay: 0.994 per frame, directions: [+1, -1]}. Each frame, advance position: position += speed × deltaTime. For each control point, calculate distance along path from pulse center. If distance < width: gaussian displacement = amplitude × exp(-pow(distance/width, 2) × 4.5) applied perpendicular to path. Pulse reflects at path endpoints (closed loop: wraps at pathLength). Multiple simultaneous pulses accumulate displacement. Pulse expires when amplitude decays below 0.5px. IDLE RETURN: Track mouse idle time. If mouse position unchanged for 5.5 seconds OR mouse leaves canvas, set interactionMode = 'returning'. All interactive displacements spring back to zero over 2.2 seconds using cubic-ease-in-out: returnProgress advances 0 to 1 over 2.2s, interactive displacement multiplied by (1 - easeInOutCubic(returnProgress)). Circle activations also return during this period. Once returned, interactionMode = 'autonomous'. Mouse movement resets idle timer and restores interactionMode = 'active'.
